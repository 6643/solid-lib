Updated names: --theme-color and --error-color

This round keeps base / raised / inset / disabled as paired surfaces, while theme-color and error-color are injected single colors. The examples below show how components consume them.

Light Mode

Base

Root background and neutral component baseline.

Raised

A lifted surface for cards and grouped content.

Inset

A pressed-in surface for inputs and embedded controls.

Disabled

Muted without disappearing, but clearly not actionable.

Theme Color

Used by the primary button and could also feed links, selection, focus, or emphasis.

Error Color

Used for validation feedback, error hints, and destructive emphasis.

Form Preview

No default border. The card floats above the page, the input sinks inward, and errors stay as a focused signal.

Dark Mode
Paper + Teal

Base

Root background and neutral component baseline.

Raised

A slightly lifted surface that stays matte, not glossy.

Inset

A more sunken surface for fields or embedded controls.

Disabled

Muted and low-energy, but still legible enough.

Theme Color

Still a single injected color. Components decide how strongly to use it.

Error Color

Bright enough to register in dark mode without becoming neon.

Form Preview

The card rises above the page, the input stays pressed in, and validation remains a local signal instead of repainting the whole field.
