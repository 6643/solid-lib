# SPA Route Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a minimal Solid 2 SPA route module with static path matching, query parsing, anchor interception, browser history sync, lazy component support, and imperative navigation.

**Architecture:** Keep a detached global route state in `src/route/state.ts`, expose a thin `Route` component that only matches `pathname`, and implement `parseParams()` as getter-backed properties linked to the global search string. Use history entry metadata instead of a separate in-memory back stack.

**Tech Stack:** TypeScript, Solid 2 beta APIs, Bun test
